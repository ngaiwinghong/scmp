For B2C — campaign briefing reference

Order summary · variable map

Each numbered piece of the card on the left is filled by a corresponding data field on the right. When briefing a new campaign, B2C only needs to provide these fields — everything else is auto-generated by the system.

1
Plan

What the user is buying

Comes from the product catalog. Doesn't change between campaigns.

plan_name: "Standard: Annual"
billing_period: "annual"
2
Offer label (the tag)

How the campaign is named

B2C picks the category. Tag text is auto-filled unless using custom, which lets B2C write up to 20 characters.

category: "custom" // 5 options
text: "Black Friday Deal"
theme: "blue" // green = free trial only
3
Summary line

Auto-generated · DO NOT WRITE

Composed from the phases below. Saves the team from writing it manually and ensures consistency.

then {p2.price} for {p2.duration}.
Renews at {p3.price}."

Phase 2 (middle row)

Discounted period

Setting compare_at enables the strikethrough. The "50% off" text is auto-computed from the comparison.

kind: "discounted"
duration_label: "Next 12 months"
price: HKD480 / year
compare_at: HKD960 // → strikethrough + auto %
6
Phase 3 (last row)

Standard rate (after promo)

Setting is_post_contract: true automatically applies the muted grey styling and hollow dot. Always include this row — it's legally required to disclose renewal price.

kind: "standard"
duration_label: "After that, every year"
is_post_contract: true // → muted
7
Descriptor under each row

Auto-generated · locked copy

Pulled from a fixed lookup table by phase kind + billing period. B2C cannot edit this copy — keeps language consistent across all campaigns.

intro × monthly →
  "Intro rate · billed monthly"
discounted × annual →
  "Discounted annual · {N}% off"

Due today

What hits the card now

Always shows the first phase's first charge. Strike-through is optional. Tax breakdown appears as a sub-line if applicable.

amount_display: "HKD30.00"
compare_at_display: "HKD85" // optional
tax_breakdown: null // only if VAT

What B2C provides

Plan ID, offer category, custom tag text (if any), and the phases — kind, duration, price, optional compare-at, optional tax.

What's auto-generated

Summary line, descriptor under each row, % off calculations, tag colour (except free trial → green), and the muted styling on post-contract rows.

What's locked

The descriptor copy library — to keep voice consistent. The strikethrough behaviour. The dot/colour rules. The card layout. Changes need design + dev review.
