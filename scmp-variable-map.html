Order summary · variable map

Each numbered piece of the card on the left is filled by a corresponding data field on the right. When briefing a new campaign, B2C only needs to provide these fields — everything else is auto-generated by the system.

Plan

What the user is buying

Comes from the product catalog. Doesn't change between campaigns.

plan_name: "Standard: Annual"
billing_period: "annual"

Offer label (the tag)

Who the offer is for

Pick a category — the tag colour follows the category automatically. The four colours signal who the offer is meant for.

// theme is derived from category

Phase 2 (middle row)

Mid-stage discounted period

Setting compare_at enables the strikethrough. The "50% off" text is auto-computed from the comparison.

Phase 3 (last row)

Standard rate (after promo)

Setting is_post_contract: true automatically applies the muted styling and hollow dot. Always include this row — disclosing renewal price is legally required.

kind: "standard"
duration: ongoing
is_post_contract: true // → muted
7
Descriptor under each row

Auto-generated · locked copy

Pulled from a fixed lookup table by phase kind + billing period. B2C cannot edit this copy — keeps language consistent across all campaigns.

intro × monthly →
  "Intro rate · billed monthly"
discounted × annual →
  "Discounted annual · {N}% off"

Duration label · locked wording

Phase labels follow a fixed template based on the number of phases. The 2-phase and 3-phase cards share the same first and last wording so the user always sees the same renewal phrasing.

2-phase card e.g. simple intro

For standard new-user offers where there are two billing periods only.

FirstFirst {X}
LastAfter that, every {X}

3-phase card stepped or trial

For stepped offers and free trials where the user moves through three billing periods.

FirstFirst {X}
MiddleNext {X}
LastAfter that, every {X}

What B2C provides

Plan ID, offer category, custom tag text (if any), and the phases — kind, duration, price, optional compare-at, optional tax.

What's auto-generated

Summary line, descriptor under each row, duration labels, % off calculations, tag colour (from category), and the muted styling on post-contract rows.

What's locked

The descriptor copy library. The duration wording. The strikethrough behaviour. The dot/colour rules. The card layout. Changes need design + dev review.
